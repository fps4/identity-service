# component-auth documentation overview

This page orients readers to the documentation for the component-auth repository. It describes what each documentation shelf contains and where to start.

## What you will find

- The **product shelf** (`docs/product/`) describes what the component does: functional specifications and product-facing references.
- The **architecture shelf** (`docs/architecture/`) describes how the component is built: technical designs and architecture decision records.

## Where to start

- New to the component? Start with the [functional specifications](specs/README.md).
- Looking for a technical decision? See the [decision records](../architecture/decisions/README.md).
- For a single view across both shelves, see the [documentation index](../README.md).
